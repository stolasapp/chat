## Design Context

### Users

Anonymous adults exploring kink/BDSM matchmaking chat.
They arrive with no account, fill a profile (gender, role,
interests), set partner filters, and get paired for 1:1 text
chat. Sessions are ephemeral. Users may be on mobile or
desktop, likely in a private context. They want to connect
quickly without friction or judgment.

### Brand Personality

**Discreet, intimate, playful.**

The interface should feel like a private room, not a public
square. Low-key confidence, not clinical sterility. A hint of
personality (the fox) without being loud.

### Logo

Line-art SVG of a fox head (front-facing, tongue out).
Tone: somewhere between provocative and playful. Used as
favicon and header mark. Monochrome, works on dark and
light backgrounds.

### Aesthetic Direction

- **Dark & intimate**: deep surfaces, muted accents, the UI
  recedes so the conversation is the focus
- **Reference**: Signal / Telegram messaging UIs. Clean,
  minimal chrome, content-first. Dense but not cluttered.
- **Anti-reference**: Omegle's bare HTML. Discord's visual
  density. Overly corporate SaaS dashboards.
- **Theme**: dark-mode primary (media query), light mode as
  secondary. Move beyond default Tailwind grays toward a
  warmer, more distinctive palette. Avoid gendered color
  associations (no pink/blue coding, no rose/mauve that
  reads feminine, no navy/steel that reads masculine). Use
  neutral warm tones (amber, ochre, warm gray) or cool
  earth tones (sage, slate, stone) that feel intentional
  and inclusive of all gender expressions.
- **Typography**: system font stack (minimal identity
  investment). Rely on weight and size hierarchy, not
  typeface variety.

### Accessibility

Enhanced (WCAG AAA target):
- Sufficient contrast ratios on dark surfaces
- `prefers-reduced-motion` support for all animations
  (typing dots, transitions)
- Keyboard navigation for all interactive elements
- Prefer semantic HTML (`<nav>`, `<main>`, `<article>`,
  `<section>`, `<dialog>`, `<fieldset>`, `<legend>`, etc.)
  over ARIA attributes; use ARIA only where semantic HTML
  is genuinely insufficient
- High contrast mode consideration
- Focus indicators visible on dark backgrounds

### Design Principles

1. **Content over chrome.** The conversation is the product.
   Every pixel of UI that isn't a message should justify its
   existence.
2. **Darkness as comfort.** Dark surfaces communicate privacy
   and intimacy. Light mode exists but dark is the default
   emotional register.
3. **Quiet confidence.** Interactions feel smooth and assured,
   never flashy. Animations are functional (typing, loading),
   not decorative.
4. **Anonymity as feature.** No avatars, no usernames, no
   history. The UI reinforces that this is a fresh, private
   encounter every time.
5. **Accessible intimacy.** Dark does not mean inaccessible.
   Contrast, focus states, and semantic markup are
   non-negotiable.
6. **Gender-neutral expression.** Colors, language, and
   layout avoid gendered associations. The interface
   welcomes all forms of identity and expression equally.
